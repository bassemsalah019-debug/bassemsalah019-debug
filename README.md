### 🛢️ Well Log Petrophysical Analysis
> *LAS File Processing · Archie Equation · OOIP/EUR Estimation · Petroleum Data Science*

<table>
<tr>
<td width="60%">

**A complete 7-stage petrophysical well log analysis pipeline** built entirely in Python using a real LAS field file — replacing expensive proprietary software (Petrel/Techlog) with a reproducible, open-source workflow. Built on domain expertise from the Energia Inc internship and Suez University graduation project.

**7-Stage Analysis Pipeline:**

| Stage | Task | Method |
|-------|------|--------|
| 1 | LAS File Ingestion | `lasio` library |
| 2 | Log QC & Curve Validation | Null detection, depth alignment |
| 3 | Vshale Calculation | GR log linear method |
| 4 | Porosity Estimation | Density / Neutron / Sonic |
| 5 | Water Saturation | **Archie Equation** Sw = √(a·Rw / φᵐ·Rt) |
| 6 | Net Pay Cutoffs | Vsh < 50%, φ > 10%, Sw < 60% |
| 7 | OOIP & EUR Estimation | Volumetric method + Recovery Factor |

</td>
<td width="40%" align="center">

**🛢️ Workflow Outputs**

| Output | Description |
|--------|-------------|
| 📉 Log Tracks | Multi-curve display |
| 📊 Crossplots | Porosity vs Sw |
| 🟩 Net Pay Flag | Cutoff-based zones |
| 💧 Sw Profile | Archie-derived |
| 🏔️ Vshale | GR-normalized |
| 🛢️ OOIP | Volumetric calc |
| 📈 EUR | Recovery estimate |

**Domain Bridge:**
Backed by real industry experience at Energia Inc — seismic, reservoir characterization & 3D Petrel workflows.

</td>
</tr>
</table>

**⚡ Why This Matters:** Combines 6 months of real reservoir characterization experience at Energia Inc + Suez University graduation project in seismic evaluation — all translated into open-source Python workflows.
